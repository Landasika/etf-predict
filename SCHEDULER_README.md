# ETF 预测系统定时任务使用说明

## 📅 功能说明

定时任务调度器会在每个交易日的特定时间点自动获取数据并发送飞书通知。

## ⏰ 定时时间点

- **上午**：9:40, 10:40, 11:40
- **下午**：13:40, 14:40

**执行条件**：
- 仅在交易日执行（周一至周五）
- 周末自动跳过
- 需要系统日期时间正确

## 🚀 使用方式

### 1. 启动 API 服务（不带定时任务）

```bash
# 前台运行
./start.sh

# 后台运行
./start.sh -s
```

### 2. 启动 API 服务 + 定时任务

```bash
# 后台运行（推荐）
./start.sh -s -d

# 前台运行（调试用）
./start.sh -d
```

### 3. 查看运行状态

```bash
# 查看服务日志
tail -f logs/server.log

# 查看调度器日志
tail -f logs/scheduler.log
```

### 4. 停止服务

```bash
./stop.sh
```

## 🔧 文件说明

### 新增文件

1. **scheduler.py** - 定时任务调度器
   - 使用 schedule 库实现定时任务
   - 判断是否为交易日
   - 调用飞书推送模块

2. **start.sh** (已修改) - 启动脚本
   - 新增 `-d` 参数：启用定时任务
   - 同时管理 API 和调度器两个进程
   - 支持前后台运行

3. **stop.sh** (已修改) - 停止脚本
   - 同时停止 API 和调度器
   - 清理所有 PID 文件

## 📋 参数说明

```bash
./start.sh [选项]

选项:
  无参数   - 前台运行 API（默认）
  -s       - 后台运行 API
  -d       - 启用定时任务（交易日的 9:40, 10:40, 11:40, 13:40, 14:40）

示例:
  ./start.sh          # 前台运行 API
  ./start.sh -s        # 后台运行 API
  ./start.sh -s -d    # 后台运行 API + 定时任务
  ./start.sh -d       # 前台运行 API + 定时任务
```

## 🔍 故障排查

### 问题1：定时任务没有执行

**检查步骤**：
1. 查看调度器日志：`tail -f logs/scheduler.log`
2. 检查进程是否存在：`ps aux | grep scheduler.py`
3. 检查是否是交易日

### 问题2：飞书消息发送失败

**检查步骤**：
1. 查看调度器日志
2. 确认 API 服务是否运行：`curl http://127.0.0.0.1:8000/api/watchlist`
3. 确认 .env 配置正确

### 问题3：缺少 schedule 库

**解决方法**：
```bash
pip3 install schedule
```

## 📝 日志文件

- **logs/server.log** - API 服务日志
- **logs/scheduler.log** - 调度器日志

## 🎯 工作流程

```
系统启动 (start.sh -s -d)
    ↓
启动 API 服务 (后台)
    ↓
启动调度器 (后台)
    ↓
等待定时时间点
    ↓
判断是否是交易日
    ↓
调用 API 获取数据
    ↓
生成 Markdown 表格
    ↓
发送到飞书
```

## ⚙️ 自定义时间点

如需修改执行时间，编辑 `scheduler.py` 中的定时设置：

```python
# 修改这些行来改变执行时间
schedule.every().day.at("09:40").do(job_send_notification)
schedule.every().day.at("10:40").do(job_send_notification)
schedule.every().day.at("11:40").do(job_send_notification)
schedule.every().day.at("13:40").do(job_send_notification)
schedule.every().day.at("14:40").do(job_send_notification)
```

## 🔔 注意事项

1. **时间准确性**：定时任务依赖系统时间，确保系统时间正确
2. **依赖安装**：需要安装 `schedule` 库
3. **交易日判断**：目前只判断周末，未包含节假日
4. **资源占用**：后台运行会占用少量内存资源
