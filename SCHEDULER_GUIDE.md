# 数据自动更新功能 - 使用指南

## 功能说明

系统现在支持**基于Python的定时数据自动更新**，无需配置Linux cron或systemd！

### 主要功能

1. ✅ **一键立即更新** - 点击按钮即可手动更新数据
2. ✅ **定时任务设置** - 通过前端界面设置每日自动更新时间
3. ✅ **进度显示** - 实时显示更新进度
4. ✅ **状态监控** - 查看调度器运行状态和下次更新时间

## 快速开始

### 1. 启动系统

```bash
cd /home/landasika/etf-predict
python run.py
```

### 2. 访问Web界面

打开浏览器访问：`http://127.0.0.1:8000`

### 3. 设置定时更新

1. 点击 **"⏰ 定时设置"** 按钮
2. 勾选 **"启用定时更新"**
3. 设置更新时间（默认 15:05）
4. 点击 **"保存设置"**

### 4. 立即更新数据

1. 点击 **"📥 立即更新数据"** 按钮
2. 确认操作
3. 等待更新完成（1-3分钟）
4. 页面会自动刷新显示最新数据

## API端点

### 获取调度器状态
```bash
GET /api/data-update/scheduler/status
```

### 配置调度器
```bash
POST /api/data-update/scheduler/configure
Content-Type: application/json

{
  "enabled": true,
  "update_time": "15:05"
}
```

### 手动触发更新
```bash
POST /api/data-update/trigger
```

## 注意事项

### Tushare Token

数据更新需要配置Tushare Token：

1. 访问 https://tushare.pro/register 注册账号
2. 获取Token
3. 编辑 `config.py`：
   ```python
   TUSHARE_TOKEN = '你的Token'
   ```

### 更新时间

- 建议设置为收盘后5-10分钟（如 15:05）
- 确保系统在该时间运行
- 只在交易日执行（周一到周五）

### 数据延迟

- Tushare数据通常在收盘后30分钟内更新
- 如果立即更新失败，可能是Tushare还未发布当天数据
- 可以稍后再试或等待定时任务自动执行

## 常见问题

### Q1: 点击"立即更新"后没有反应？

**A**: 检查浏览器控制台是否有错误，确保：
- 系统已启动
- API地址正确
- 网络连接正常

### Q2: 更新失败，提示TUSHARE_TOKEN未配置？

**A**:
1. 注册Tushare账号
2. 在 `config.py` 中设置 `TUSHARE_TOKEN`
3. 重启系统

### Q3: 定时任务没有执行？

**A**: 检查：
1. 调度器是否已启用（点击"定时设置"查看）
2. 系统是否在该时间运行
3. 查看日志：`logs/data_update_scheduler.log`

### Q4: 更新后数据还是旧的？

**A**:
1. 强制刷新浏览器：`Ctrl + Shift + R`
2. 点击"🔄 刷新页面"按钮
3. 检查"数据更新日期"列是否已更新

### Q5: 如何停止定时任务？

**A**:
1. 点击"⏰ 定时设置"
2. 取消勾选"启用定时更新"
3. 点击"保存设置"

## 技术架构

### 调度器实现
- 使用Python `schedule` 库
- 运行在独立线程中
- 每分钟检查一次待执行任务

### 更新流程
1. 触发更新（手动或定时）
2. 从watchlist获取ETF列表
3. 调用Tushare API下载数据
4. 更新SQLite数据库
5. 清理批量缓存
6. 更新完成

### 并发控制
- 更新任务在后台线程执行
- 不阻塞API服务器
- 支持通过API查询状态
