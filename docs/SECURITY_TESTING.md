# ETF预测系统 - 安全测试指南

## 测试环境准备

### 1. 配置环境变量

```bash
# 创建 .env 文件（如果不存在）
cat > .env << EOF
SESSION_SECRET_KEY=test-secret-key-for-development-only
AUTH_KEY=test123
EOF
```

### 2. 启动系统

```bash
python run.py
```

系统将在 http://127.0.0.1:8000 启动

## 安全测试

### 测试 8: 防暴力破解 - IP限制

**步骤:**
1. 连续输入错误秘钥 6 次
2. 第 6 次尝试

**预期结果:**
- 前 5 次: 显示"秘钥错误，请重试"
- 第 6 次: 显示"登录尝试次数过多，请在 XXX 秒后重试"
- 无法继续尝试

**实际结果:** ⬜

---

### 测试 9: 锁定时间验证

**步骤:**
1. 触发IP锁定（连续失败5次）
2. 等待 15 分钟
3. 再次尝试登录

**预期结果:**
- 15分钟后锁定解除
- 可以正常尝试登录

**实际结果:** ⬜

---

### 测试 10: 会话劫持防护

**步骤:**
1. 登录系统
2. 打开浏览器开发者工具 → Application → Cookies
3. 查看session cookie

**预期结果:**
- Cookie设置了`HttpOnly`属性
- Cookie设置了`SameSite=lax`属性
- Session值是随机的，不易猜测

**实际结果:** ⬜

---

### 测试 11: 跨站请求伪造(CSRF)防护

**步骤:**
1. 登录系统
2. 在另一个标签页尝试通过表单POST到`/logout`

**预期结果:**
- SameSite=lax设置提供基础CSRF保护
- 正常场景下可以登出

**实际结果:** ⬜

---

### 测试 12: 时序攻击防护

**步骤:**
1. 准备两个测试秘钥：
   - 正确秘钥: `test123`
   - 错误秘钥: `wrongkey`
2. 分别测量登录响应时间（多次取平均）

**预期结果:**
- 使用`hmac.compare_digest`函数
- 两种秘钥的验证时间基本相同（差异<1ms）

**实际结果:** ⬜

## 测试工具推荐

### 1. 浏览器开发者工具
- 查看Cookie和会话信息
- 检查网络请求和响应

### 2. curl (命令行测试)
```bash
# 测试未登录访问API
curl -i http://127.0.0.1:8000/api/watchlist

# 测试登录
curl -c cookies.txt -X POST -d "auth_key=test123" http://127.0.0.1:8000/login

# 使用session访问API
curl -b cookies.txt http://127.0.0.1:8000/api/watchlist
```

### 3. Python脚本测试
```python
import requests

# 测试登录
session = requests.Session()
response = session.post("http://127.0.0.1:8000/login", data={"auth_key": "test123"})
print(f"登录状态: {response.status_code}")

# 测试API访问
response = session.get("http://127.0.0.1:8000/api/watchlist")
print(f"API状态: {response.status_code}")
```
