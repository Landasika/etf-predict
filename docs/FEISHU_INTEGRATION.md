# 飞书推送集成使用指南

## 🎯 功能概述

飞书推送已完全集成到ETF预测系统中，无需单独运行脚本。所有配置都可以在Web界面的设置页面完成。

---

## 📱 配置步骤

### 1. 获取飞书机器人信息

#### 步骤1: 创建飞书应用
1. 访问 [飞书开放平台](https://open.feishu.cn/)
2. 创建企业自建应用
3. 获取 `App ID` 和 `App Secret`

#### 步骤2: 获取权限
在应用权限页面开启以下权限：
- `im:message` - 发送消息权限
- `im:chat` - 读取会话权限

#### 步骤3: 获取 Chat ID
1. 在飞书中创建或选择一个群聊
2. 添加机器人到群聊
3. 群聊 → 设置 → 群信息 → 群ID (格式: `oc_xxxxxxxx`)

### 2. 在系统中配置

1. **打开设置页面**
   - 访问 http://127.0.0.1:8000/settings
   - 找到"📱 飞书通知配置"部分

2. **启用飞书通知**
   - 打开"启用飞书通知"开关

3. **添加机器人**
   - 点击"➕ 添加机器人"
   - 填写机器人信息：
     * 名称: 如"默认机器人"
     * App ID: `cli_xxxxxxxxx`
     * App Secret: 应用密钥
     * Chat ID: `oc_xxxxxxxx`
   - 可添加多个机器人用于不同用途

4. **选择通知类型**
   - ✅ 策略信号提醒 - 买卖信号时推送
   - ⬜ 数据更新通知 - 数据更新完成时推送
   - ⬜ 回测完成通知 - 回测完成时推送
   - ✅ 错误告警 - 系统错误时推送

5. **测试连接**
   - 在"测试连接"部分选择机器人
   - 点击"发送测试"按钮
   - 查看飞书群是否收到测试消息

6. **保存设置**
   - 点击页面底部的"💾 保存设置"按钮

---

## 🔧 通知类型说明

### 策略信号提醒
当策略产生买卖信号时自动推送消息：
```
🟢 策略信号提醒

ETF: 沪深300ETF (510330.SH)
策略: MACD激进策略
信号: BUY

时间: 2026-03-10 14:30:00
```

### 数据更新通知
市场数据更新完成时推送：
```
✅ 数据更新完成

更新数量: 52 个ETF
时间: 2026-03-10 15:05:00
```

### 错误告警
系统发生错误时推送：
```
⚠️ 系统错误告警

错误类型: 数据更新失败
错误信息: 连接超时
时间: 2026-03-10 10:30:00

请检查系统日志
```

## 🤖 多机器人管理

### 使用场景

1. **默认机器人** - 所有通知都发送到这个机器人
2. **专用机器人** - 不同类型的通知发送到不同群聊
   - 信号群: 只接收交易信号
   - 运维群: 只接收系统错误
   - 更新群: 只接收数据更新通知

### 配置示例

```
机器人1: "交易信号群"
- 通知: ✅ 策略信号提醒
- 用途: 实时推送买卖信号

机器人2: "系统运维群"
- 通知: ✅ 错误告警
- 用途: 监控系统运行状态

机器人3: "数据更新群"
- 通知: ✅ 数据更新通知
- 用途: 数据更新提醒
```

## 🔍 故障排除

### 问题1: 测试消息发送失败

**可能原因:**
- App ID 或 App Secret 错误
- Chat ID 格式不正确
- 机器人未添加到群聊
- 应用未授权相应权限

**解决方案:**
1. 检查配置信息是否正确
2. 确认机器人已添加到群聊
3. 检查应用权限是否已开启
4. 查看服务器日志: `tail -f logs/*.log`

### 问题2: 收不到策略信号

**可能原因:**
- "策略信号提醒"未启用
- 飞书通知总开关未启用
- 机器人被禁用

**解决方案:**
1. 检查"通知类型"中的"策略信号提醒"是否勾选
2. 检查"启用飞书通知"开关是否打开
3. 检查机器人卡片中的开关是否打开

### 问题3: 消息延迟

**可能原因:**
- 网络延迟
- 飞书API限流

**解决方案:**
- 正常现象，通常延迟在1-5秒内
- 如延迟超过10秒，检查网络连接

## 💡 使用建议

### 开发/测试环境
- 使用单独的测试群
- 仅启用"策略信号提醒"
- 关闭"数据更新通知"避免打扰

### 生产环境
- 配置多个机器人分群管理
- 启用所有通知类型
- 定期检查配置有效性
- 监控推送频率避免限流

### 安全建议
1. **保护 App Secret**
   - 不要将密钥提交到版本控制
   - 定期更换密钥

2. **权限最小化**
   - 只开启必要的权限
   - 使用专用群聊而非全员群

3. **消息频率控制**
   - 避免过于频繁的推送
   - 合理设置通知类型
