# ETF预测系统 - 身份认证使用指南

## 概述

系统已启用基于秘钥的身份认证，所有用户必须输入正确的访问秘钥才能使用系统。

## 快速开始

### 1. 生成安全密钥

```bash
# 生成会话密钥
python -c "import secrets; print(secrets.token_urlsafe(32))"

# 输出示例: abc123xyz456... (保存此值)
```

### 2. 配置环境变量

创建 `.env` 文件（或编辑现有的）:

```bash
# 会话密钥（使用上面生成的值）
SESSION_SECRET_KEY=your-generated-secret-key

# 访问秘钥（设置你的访问密码）
AUTH_KEY=your-access-password
```

### 3. 启动系统

```bash
python run.py
```

### 4. 登录系统

1. 访问 http://127.0.0.1:8000
2. 自动跳转到登录页面
3. 输入你设置的 `AUTH_KEY`
4. 登录成功后可正常使用所有功能

## 功能特性

### 会话管理
- **有效期**: 浏览器关闭时会话失效
- **安全性**: 使用随机会话ID，防止会话固定攻击
- **跳转**: 登录后自动跳转到原访问页面

### 防暴力破解
- **限制**: 同一IP 5分钟内最多尝试5次
- **锁定**: 超过限制后锁定15分钟
- **记录**: 所有登录尝试都会被记录

### 安全措施
- **SHA256哈希**: 秘钥使用哈希存储，防止明文泄露
- **时序攻击防护**: 使用 `hmac.compare_digest` 防止计时攻击
- **CSRF保护**: Cookie 设置 SameSite 属性
- **HTTPS支持**: 生产环境建议启用 HTTPS

## 默认配置

如果未设置环境变量，系统使用以下默认值：

```python
SESSION_SECRET_KEY = 'change-this-in-production-please-use-env-var'
AUTH_KEY = 'admin123'
```

⚠️ **警告**: 生产环境请务必修改这些默认值！

## 使用说明

### 登录
访问任意页面会自动跳转到登录页，输入秘钥即可。

### 登出
点击页面右上角的 "🚪 登出" 按钮。

### 修改秘钥
1. 编辑 `.env` 文件中的 `AUTH_KEY`
2. 重启系统: `python run.py`

## 故障排除

### 登录失败
- 检查秘钥是否正确
- 检查是否被锁定（等待15分钟或重启服务器）
- 查看服务器日志了解详细错误

### 会话过期
- 正常现象，浏览器关闭后需要重新登录
- 如需延长会话，修改 `core/auth.py` 中的 `max_age` 参数

### 找不到登录页面
- 确保服务器正在运行
- 检查是否正确访问 http://127.0.0.1:8000

## 生产部署建议

1. **使用HTTPS**: 在反向代理（如Nginx）中配置SSL证书
2. **强密钥**: 使用至少32位的随机字符串作为 `AUTH_KEY`
3. **环境变量**: 永远不要将 `.env` 文件提交到版本控制
4. **日志监控**: 定期检查登录日志，发现异常行为
5. **定期轮换**: 建议3-6个月更换一次访问秘钥

## 安全注意事项

- 不要使用默认秘钥 `admin123`
- 不要在公开场合分享访问秘钥
- 定期更换会话密钥和访问秘钥
- 监控登录日志，发现异常及时处理
- 生产环境务必配置 HTTPS
