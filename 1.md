# MACD激进策略 - 仓位管理与买卖规则详解

## 策略概述

MACD激进策略是基于MACD指标的趋势跟踪策略，通过信号强度来决定仓位大小（0-10仓），并配合止损、止盈、信号反转等机制进行风险管理。

---

## 一、信号生成系统

### 1.1 MACD指标参数
- **快线(DIF)周期**: 8（默认）
- **慢线(DEA)周期**: 17（默认）
- **信号线周期**: 5（默认）

### 1.2 信号类型与强度

策略根据不同的市场形态生成买卖信号，每个信号都有对应的强度值（-10到+10）：

#### 买入信号（强度 6, 7, 8, 9, 10）

| 信号类型 | 强度 | 触发条件 | 说明 |
|---------|------|---------|------|
| **回踩MA60未破+MACD金叉** | 10 | 价格回踩MA60未破 + MACD金叉 | 最强买入信号 |
| **正鸭嘴形态** | 9 | DIF/DEA将死不死再度开口 + 在零轴上方 | 强势形态 |
| **零轴上方金叉(空中加油)** | 8 | DIF上穿DEA + DIF>0 | 趋势中继 |
| **底背离** | 7 | 价格新低+MACD未新低 + 确认突破 | 反转信号 |
| **普通MACD金叉** | 6 | DIF上穿DEA | 标准金叉 |

**注意**: MACD激进策略**不产生1-5强度的买入信号**

#### 卖出信号（强度 -6, -7, -8, -9）

| 信号类型 | 强度 | 触发条件 | 说明 |
|---------|------|---------|------|
| **倒鸭嘴形态** | -9 | DIF/DEA将金不金再度向下 + 在零轴下方 | 强势卖出 |
| **零轴下方死叉(加速下跌)** | -8 | DIF下穿DEA + DIF<0 | 加速下跌 |
| **顶背离** | -7 | 价格新高+MACD未新高 + 确认跌破 | 反转卖出 |
| **普通MACD死叉** | -6 | DIF下穿DEA | 标准死叉 |

**注意**: MACD激进策略**不产生-10和-1到-5强度的卖出信号**

#### 辅助过滤（增强信号强度）

- **MA60支撑**: 买入时价格在MA60上方，信号强度+2
- **MA60压力**: 卖出时价格在MA60下方，信号强度-2

## 二、仓位管理系统

### 2.1 资金划分

总资金分为 **10个仓位**，每个仓位价值 = 初始资金 ÷ 10

例如：初始资金 ¥2,000，每仓 = ¥200

### 2.2 买入规则（开仓/加仓）

买入优先级：**PRIORITY 4**（在止损、止盈、信号反转之后）

#### 信号强度与目标仓位

| 信号强度 | 目标仓位 | 说明 |
|---------|---------|------|
| 6 | 5仓 | 标准金叉 |
| 7 | 6仓 | 底背离 |
| 8 | 7仓 | 零轴上方金叉 |
| 9 | 8仓 | 正鸭嘴形态 |
| 10 | 10仓（满仓） | 回踩MA60未破+金叉 |

**注意**：
- 实际信号强度只有 **6, 7, 8, 9, 10**
- 没有强度1-5的信号，所以不会出现1-4仓的买入
- 代码中1-3的逻辑是历史遗留或为其他策略保留

#### 实际买入逻辑

```python
# 计算目标仓位（实际只有6-10的信号）
if signal_strength <= 6:
    desired_positions = signal_strength - 1  # 6→5仓
elif signal_strength <= 9:
    desired_positions = signal_strength      # 7→7仓, 8→8仓, 9→9仓
else:
    desired_positions = 10                   # 10→满仓

# 实际映射：
# 强度 6 → 5仓
# 强度 7 → 6仓
# 强度 8 → 7仓
# 强度 9 → 8仓
# 强度 10 → 10仓（满仓）

# 计算新增仓位
positions_to_add = desired_positions - positions_used
positions_to_add = min(positions_to_add, 10 - positions_used)

# 执行买入
if positions_to_add > 0 and cash >= investment:
    shares = int(investment // price)
    # 买入并更新加权平均成本
```

#### 关键特性

1. **分批建仓**: 同一方向信号可以持续加仓
2. **加权平均成本**: 多次买入使用加权平均计算成本价
3. **T+1执行**: 信号延迟1天执行（避免未来函数）

## 三、卖出系统（4级优先级）

### 3.1 PRIORITY 1: 止损（风险控制第一）

**触发条件**: 亏损 >= 10%

**执行动作**: **全部清仓**

```
if pnl_pct <= -10%:
    sell(all_positions)
    重置所有状态
```

### 3.2 PRIORITY 1.5: 追踪止盈（可选）

**触发条件**:
1. 盈利 >= 15% 后激活
2. 从最高点回撤 >= 5%

**执行动作**: **卖出50%仓位**

```
if pnl_pct >= 15% and (highest_price - price) / highest_price >= 5%:
    sell(50%)
    重置最高价
```

### 3.3 PRIORITY 2: 分批止盈（锁定利润）

**分三批止盈**:

| 批次 | 盈利阈值 | 卖出比例 | 说明 |
|-----|---------|---------|------|
| 第一批 | 15% | 30% | 锁定部分利润 |
| 第二批 | 30% | 30% | 继续锁定 |
| 第三批 | 35% | 40% | 清仓止盈 |

**执行逻辑**:

```python
# 第一批止盈
if pnl_pct >= 15% and not take_profit1_triggered:
    sell(30%)
    take_profit1_triggered = True

# 第二批止盈
elif pnl_pct >= 30% and not take_profit2_triggered:
    sell(30%)
    take_profit2_triggered = True

# 第三批止盈
elif pnl_pct >= 35% and not take_profit3_triggered:
    sell(40%)  # 全部清仓
    take_profit3_triggered = True
```

### 3.4 PRIORITY 3: 信号反转（趋势恶化）

**触发条件**: signal_strength < 0（卖出信号）

**卖出数量**（根据信号强度）:

| 信号强度 | 卖出仓位 | 说明 |
|---------|---------|------|
| -6 | 3-5仓 | 标准死叉减仓 |
| -7 | 4-6仓 | 顶背离减仓 |
| -8 | 5-7仓 | 零轴下死叉大幅减仓 |
| -9 | 8-10仓 | 倒鸭嘴清仓 |

**注意**: 代码中的逻辑是通用的，但MACD激进策略实际只产生-6到-9的信号

```python
if signal_strength < 0:
    if signal_strength >= -3:  # 实际不会触发
        positions_to_close = min(positions_used, 1 + abs(signal_strength))
    elif signal_strength >= -6:
        positions_to_close = min(positions_used, 3 + abs(signal_strength) - 3)
        # -6: 3+6-3 = 6, 但受positions_used限制
    else:  # -7, -8, -9
        positions_to_close = positions_used  # 全部清仓
```

**实际情况**：
- 强度-6: 卖出约3-5仓（不一定是全部）
- 强度-7到-9: **全部清仓**

## 四、交易流程图

```
每天开盘:
┌─────────────────────────────────────┐
│  1. 检查止损 (亏损>=10%)             │
│     ├─ 是 → 全部清仓，重置状态        │
│     └─ 否 → 继续                     │
├─────────────────────────────────────┤
│  2. 检查追踪止盈 (可选)              │
│     ├─ 触发 → 卖出50%               │
│     └─ 未触发 → 继续                 │
├─────────────────────────────────────┤
│  3. 检查分批止盈 (15%/30%/35%)      │
│     ├─ 第一批(15%) → 卖出30%         │
│     ├─ 第二批(30%) → 卖出30%         │
│     ├─ 第三批(35%) → 卖出40%（清仓）  │
│     └─ 未触发 → 继续                 │
├─────────────────────────────────────┤
│  4. 检查信号反转 (signal_strength<0) │
│     ├─ 是 → 根据强度卖出对应仓位     │
│     └─ 否 → 继续                     │
├─────────────────────────────────────┤
│  5. 检查买入信号 (signal_strength>0) │
│     ├─ 是 → 根据强度买入对应仓位     │
│     └─ 否 → 持仓不变                 │
└─────────────────────────────────────┘
```

## 六、策略特点

### 优势

1. **趋势跟踪**: MACD指标适合捕捉中期趋势
2. **分批建仓**: 根据信号强度逐步建仓，降低风险
3. **多级止盈**: 15%/30%/35%分批锁定利润
4. **严格止损**: 10%止损保护本金
5. **信号反转**: MACD死叉及时减仓/清仓

### 劣势

1. **震荡市场表现差**: MACD在横盘震荡中容易产生假信号
2. **滞后性**: MACD是趋势指标，信号有一定滞后
3. **手续费损耗**: 频繁交易会产生手续费（卖出0.5%）
4. **回撤较大**: 趋势反转时可能产生较大回撤

### 适用场景

- ✅ **趋势明显的牛市/熊市**
- ✅ **波动率适中的市场**
- ❌ **横盘震荡市场**
- ❌ **极端波动市场**

## 八、示例交易

### 示例1: 完整趋势交易

```
Day 1: MACD零轴上方金叉（强度8）
  → 买入7仓（¥1400）

Day 5: 盈利达到16%（触发第一批止盈）
  → 卖出30%（2仓），剩余5仓

Day 10: 盈利达到31%（触发第二批止盈）
  → 卖出30%（1.5仓），剩余3.5仓

Day 15: 盈利达到36%（触发第三批止盈）
  → 卖出40%（剩余全部，3.5仓）
  → 清仓，锁定利润

总收益: 约31%（扣除手续费）
```

### 示例2: 止损保护

```
Day 1: MACD金叉（强度6）
  → 买入5仓（¥1000）

Day 3: 价格下跌，亏损达到11%（触发止损）
  → 全部清仓（5仓）
  → 亏损约11.5%（含手续费）

止损保护: 避免进一步亏损
```

### 示例3: 信号反转

```
Day 1: MACD金叉（强度7）
  → 买入7仓（¥1400）

Day 5: 盈利5%，未触发止盈

Day 6: MACD死叉（强度-6）
  → 卖出5仓（根据强度-6）
  → 剩余2仓

Day 8: 再次死叉（强度-8）
  → 清仓剩余2仓

结果: 小幅盈利或小幅亏损，避免深度套牢
```

## 十、总结

MACD激进策略的核心思想是：

1. **趋势为王**: MACD金叉买入，死叉卖出
2. **仓位管理**: 信号强度决定仓位大小（1-10仓）
3. **风险控制**: 10%止损保护本金
4. **利润锁定**: 15%/30%/35%分批止盈
5. **灵活应对**: 信号反转及时减仓/清仓

策略适合趋势明显的市场，在震荡市场中可能会频繁止损。建议配合其他指标（如MA60、布林带）进行过滤，提高胜率。
